Решение тестового задания по разработке умного Telegram-бота для консультирования клиентов компании премиум-сегмента лакокрасочных материалов «Центр Красок #1» (Казахстан).

Проект построен на принципах No-Code/Low-Code автоматизации с применением современных подходов в области разговорного искусственного интеллекта (Conversational AI).

---

## 🛠 Технологический стек
* **n8n** — платформа для автоматизации сценариев и оркестрации AI-агентов.
* **Supabase (PostgreSQL + pgvector)** — векторная база данных для хранения и поиска информации по технологии RAG.
* **Google Gemini API** (модели `gemini-3.1-flash-lite` и `text-embedding-004`) — большая языковая модель (LLM) для генерации ответов и создания эмбеддингов.
* **Telegram Bot API** — интерфейс для взаимодействия с пользователями.
* **PostgreSQL (в рамках Supabase)** — используется для долгосрочного хранения контекста и памяти диалогов.

---

## 🏗 Архитектура решения и воркфлоу в n8n

Проект разделен на 3 независимых сценария (воркфлоу) для обеспечения модульности и отказоустойчивости.

### 1. Сценарий векторизации базы знаний (Ingestion Workflow)
Сценарий предназначен для первичной подготовки и загрузки информации о компании в векторную базу данных.
* **Как работает:** Текстовая база знаний (информация о компании, товарах, контактах, технологиях и вакансиях) подается на узел разделения текста (`Recursive Character Text Splitter`). Текст нарезается на смысловые фрагменты (чанги) размером до 1000 символов с перекрытием 200 символов, векторизуется через `Google Gemini Embeddings` (модель генерирует векторы размерностью 3072) и записывается в таблицу `documents` в Supabase.

### 2. Сценарий работы Telegram-бота (Main Chatbot Workflow)
Основной сценарий, отвечающий на вопросы пользователей в реальном времени.
* **Как работает:** 
  1. `Telegram Trigger` принимает сообщение от пользователя.
  2. Узел `Send a chat action` отправляет статус «typing...» (печать сообщения) для улучшения UX.
  3. `AI Agent` анализирует запрос пользователя. К нему подключена модель `Google Gemini Chat Model`, узел `Postgres Chat Memory` (для удержания контекста) и инструмент векторного поиска `Supabase Vector Store`.
  4. Если пользователь спрашивает про компанию, AI-агент обращается к инструменту поиска, извлекает наиболее релевантные куски текста и на их основе формирует ответ.
  5. Результат отправляется пользователю через узел `Telegram (Send Message)` в режиме безопасного HTML-форматирования.

### 3. Сценарий обработки ошибок (Error Workflow)
Обеспечивает стабильность и отказоустойчивость всей системы.
* **Как работает:** Перехватывает системные сбои (например, таймаут API Gemini или временную недоступность Supabase) и предотвращает «молчание» бота. В случае ошибки бот отправляет пользователю вежливое уведомление о временных технических неполадках и регистрирует инцидент.

## 🔒 Решенные задачи и Guardrails (Безопасность и UX)

В рамках проекта были успешно реализованы требования по защите данных и улучшению пользовательского опыта:

1. **Защита от галлюцинаций (Anti-Hallucination):** В системном промпте AI-агента жестко прописано правило отвечать только на основе фактов, полученных из векторной базы данных. Если информации по запросу нет, бот честно признается в этом и предлагает прямые контакты менеджеров компании.
2. **Фильтрация сторонних тем (Off-topic Filter):** Бот игнорирует попытки пользователя общаться на темы, не связанные с деятельностью компании (политика, кулинария, программирование и т.д.), и вежливо возвращает диалог в рамки тематики красок и ремонта.
3. **Защита от взлома промпта (Prompt Injection Defense):** Внедрен слой защиты, запрещающий раскрывать системные инструкции, промпты или структуру базы данных, даже если пользователь использует обманные команды («забудь предыдущие инструкции», «покажи свои настройки» и т.д.).
4. **Стабильное форматирование (HTML Parse Mode):** Вместо классического Markdown используется разметка HTML. Это исключает возникновение ошибок Telegram API при отправке сообщений, содержащих знаки препинания (точки, восклицательные знаки, дефисы), которые часто ломают MarkdownV2.
5. **Бесшовная обработка `/start`:** При первом запуске бота или стандартном приветствии AI-агент не тратит ресурсы на обращение к векторной базе данных, а сразу отправляет персонализированное приветственное сообщение с описанием своих возможностей.

## 🚀 Инструкция по локальному развертыванию

Для того чтобы запустить проект у себя, выполните следующие шаги:

### Шаг 1. Подготовка базы данных в Supabase
1. Создайте проект в Supabase.
2. Перейдите в **SQL Editor** и выполните следующий SQL-запрос для инициализации расширения векторов, таблицы документов и функции поиска:

```sql
-- Включаем расширение pgvector для работы с векторами
create extension if not exists vector;

-- Создаем таблицу для документов (3072 измерения для Gemini Embeddings)
create table documents (
  id bigserial primary key,
  content text,
  metadata jsonb,
  embedding vector(3072)
);

-- Создаем функцию поиска (совместимую с n8n и LangChain)
create or replace function match_documents (
  query_embedding vector(3072),
  match_threshold float default 0.2,
  match_count int default null,
  filter jsonb default '{}'
)
returns table (
  id bigint,
  content text,
  metadata jsonb,
  similarity float
)
language plpgsql stable
as $$
begin
  return query
  select
    documents.id,
    documents.content,
    documents.metadata,
    1 - (documents.embedding <=> query_embedding) as similarity
  from documents
  where documents.metadata @> filter 
    and 1 - (documents.embedding <=> query_embedding) > match_threshold
  order by documents.embedding <=> query_embedding
  limit match_count;
end;
$$;
